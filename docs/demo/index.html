Guide

To calculate a single categoryScore, the main principle is the difference between the categoryTarget and the actual categoryValue. Things to note for the user adjustable properties are as follows:

  • The categoryValue can not be dynamically adjusted by the front end user, because it is a measured value in open world.

  • The categoryTarget is adjustable, with the min or max value automatically being set to the categoryValue. Should the categoryValue>= categoryTarget, then the categoryDirection should be set to 1.
    (E.g. the category goal is it to optimize the measured process for as little time as possible). If set correctly the categoryTarget cannot be set bigger than the categoryValue. The Result is the adjustable categoryTarget will automatically stop at max = categoryValue.
    Now The other way around: Should the categoryValue <= categoryTarget, then the categoryDirection should be set to -1 . (E.g. the category goal is it to optimize the measured process for the hightest efficiency possible). If set correctly the categoryTarget cannot be set smaller than the categoryValue. The Result is the adjustable categoryTarget will automatically stop at min = categoryValue.

    The min and max values for the range sliders are estimated with the following assumption:

    1. if the categoryUnit is "%":

      Depending on the categoryDirection the min is set to "0" or max is set to "100"

    2. if the categoryUnit is not "%":

      Lets say our value should always be larger than our categoryTarget, so therefore the categoryDirection is "1". This also already defines the max value to categoryValue. The min value is then "guessed" by calculating min = categoryValue − 5 ⋅ categoryGrain This assumes that the measured categoryValue is already as low as possible, so a factor larger than 5 makes no sense. Reducing things to some amount, generally speaking, tend to have an absolute limit, which can be reached.
      This changes when the categoryValue is lower than the categoryTarget, so therefore the categoryDirection is "-1". Since we already know the min value, which is the categoryValue, only the max value needs to be calculated. max = categoryValue + 14 ⋅ categoryGrain Here is it the other way around. Trying to increase a value can be almost limitless with no definable limit, so therefore a factor of 14 was used.

  • The categoryBias is adjustable, with the categoryBias accepting any Integer between -5 and 5. If the categoryBias is grater than 0, the category will experience a favored rating method and therefore giving it a better score.
    If the categoryBias is smaller than 0, the category will experience a unfavored rating method and therefore giving it a lower score.
    Once the categoryBias reaches the upper or lower limit the score tries to be the as close to the hightest or lowest score possible, this beeing 1000/1000 or 0/1000

  • The categoryWeight can be adjusted, which affects the overall score, but not the individual categoryScore! When adjusting one categoryWeight all the other Weights will adjust accordingly, so all percentages added up together will always be 100%.

    ∑ categoryNumber = 0 categoryCount categoryScore categoryNumber 1000 = 1 = 100 %

    For ease of use the lowest individual categoryWeight is 0.01 %. This ensures, that no single categoryWeight will be stuck at 0%.
    Due to the nature of using linear Algebra (more specific Matrix calculations) for calculation the weights. Depending on the number of categories, if one category reaches 0% it will stay at 0% and the other weight will compensate, which is an unwanted effect
    To avoid errors don't leave the categoryWeight value empty, and instead set a preset.

Things to note for the static properties are as follows:

  • categoryNumber:
    This is important to keep Track of the different categories. Remember to stat with 0 when numbering the categories.

  • categoryNames:
    These will be displayed in the category column as the category name.

  • categoryUnits:
    These are the units which should be displayed

  • categoryTargets:
    Remember this a dynamically adjustable value. But you must give a preset for initialization. Also make sure the values are in line with the categoryDirection and the categoryValues

  • categorySteps:
    This will set the step of the all target value range sliders. Basically by how much fine control of the slide is wanted.

  • categoryDirections:
    This defines, whether the categoryValue is trying to reach a value greater than it self, in which case the categoryDirection should be set to "-1", or if the categoryValue is trying to reach a categoryTarget which is smaller than itself, in which case the categoryDirection should be set to "1".
    Any other values other than "-1" or "1" are strictly forbidden.
    NOTE: If the category value is larger than its target (more ---> positive ---> +) the direction should be set to "1"
    If the category value is lower than its target (less --> negative ---> -) the direction should be set to "-1"

  • categoryGrains:
    With the categoryGrain the dimensions of the numbers can be tuned to the category need. The categoryGrain should be set to the minimal relevant increment or decrement the categoryValue has relevancy to. Another thing, which might help is this equation, which states, that the categoryScore is always "500", when the categoryValue is equals to the categoryGrain times the categoryDirection plus the times the categoryTarget times the squared categoryDirection.
